=================================

Local development (Docker) — start here
---------------------------------------
Develop and test workflows on local n8n, then export JSON to prod.

Your local n8n already runs via **n8n-test-project** (not a second Docker stack).

1. Start local stack (if not running):
     cd ~/src/n8n-test-project
     docker compose up -d

2. Open editor: http://localhost:5678

3. Workflow **BCAI Website Chatbot (Ask BCAI)** should appear after import.
   If missing, import from repo:
     docker cp ~/src/web_site/n8n/bcai-website-chatbot.workflow.json n8n:/tmp/w.json
     docker exec n8n n8n import:workflow --input=/tmp/w.json
   Or: Workflows → Import from file → `n8n/bcai-website-chatbot.workflow.json`
   (Do not import on top of an existing copy — delete the old workflow first.)

4. **Credentials** (same keys as prod; OAuth redirect URI differs for local):
     - **OpenRouter API** → your OpenRouter key
     - **Google Drive OAuth2** → sign in; add redirect URI in Google Cloud:
       http://localhost:5678/rest/oauth2-credential/callback

5. On **When chat message received** → Allowed Origins already includes localhost
   in the repo JSON for local widget tests.

6. **Save** → **Activate** → copy webhook URLs from nodes:
     Chat:  http://localhost:5678/webhook/bcai-website-chat/chat
     Warm:  http://localhost:5678/webhook/bcai-warm-knowledge

7. Test in n8n (no website needed):
     - Open workflow → **Chat** panel at bottom, or
     - curl warm cache:
         curl -sS -X POST http://localhost:5678/webhook/bcai-warm-knowledge \\
           -H 'Content-Type: application/json' -d '{}'
     - curl chat:
         curl -sS -X POST http://localhost:5678/webhook/bcai-website-chat/chat \\
           -H 'Content-Type: application/json' \\
           -d '{"action":"sendMessage","chatInput":"What services do you offer?","sessionId":"local-test"}'

8. Test with live website widget (optional):
     Temporarily set in `js/site-config.js`:
       chatWebhookUrl: 'http://localhost:5678/webhook/bcai-website-chat/chat'
       chatWarmCacheUrl: 'http://localhost:5678/webhook/bcai-warm-knowledge'
     Serve site locally: `python3 -m http.server 8080` from repo root.
     Revert URLs before deploying to VPS.

9. When happy → **Export** workflow from n8n → overwrite
      `n8n/bcai-website-chatbot.workflow.json` → commit → **prod clean import**
      (delete old prod workflow, import JSON, re-attach credentials, activate).

Knowledge cache (10 minutes)
----------------------------
The workflow caches merged Drive knowledge in workflow static data for 10 minutes.

Flow:
  Chat Trigger (streaming) → Check knowledge cache → Cache hit?
    - YES → Prepare Ask BCAI context → Ask BCAI Agent (+ OpenRouter Chat Model)
    - NO  → List → Download → Build knowledge context → Warm only?
              → NO  → Prepare Ask BCAI context → Ask BCAI Agent → …
              → YES → (warm path only; should not happen on chat)

  Warm knowledge cache webhook (panel open on website):
    POST /webhook/bcai-warm-knowledge
    → Check warm cache → Warm cache fresh?
         YES → Format warm cache response { ok, skipped: true }
         NO  → List → Download → Build → Warm only? → Warm cache complete → response

Both triggers share the same 10-minute static cache and the same List/Download nodes.

After you update markdown files in Drive, wait up to 10 minutes for the cache
to refresh — or deactivate/reactivate the workflow to clear it immediately.

To change TTL: edit **Check knowledge cache** node — CACHE_TTL_MS (default 600000).

Note: use $input.first().json in Check knowledge cache (not $('When chat message received')).
Do not "Execute step" on Check cache alone — run the full workflow from the chat panel,
or execute from the Chat Trigger node so input is present.

Google Drive — List knowledge files node
----------------------------------------
Do NOT use Custom API Call or Search File/Folder Name with an empty query.

Configure manually (required — empty Search Query returns random Drive files):
  Resource:     File/Folder
  Operation:    Search
  Search Method: Advanced Search          ← NOT "Search File/Folder Name"
  Query String: 'FOLDER_ID_HERE' in parents and trashed = false and mimeType != 'application/vnd.google-apps.folder'
  What to search: Files
  Return All:   ON

Get FOLDER_ID_HERE from the folder URL in Drive:
  https://drive.google.com/drive/folders/XXXXXXXX  → use XXXXXXXX

Current knowledge folder name: chat_bot_repo (7 markdown files).
Original folder ID was 11K4jmUI8SZMqNJbhrwnBU63x9tE8EkmD — use whichever folder you upload to.

Expected output: ~7 items (website-home.md, bot-guardrails.md, etc.) — NOT 50 "Untitled" folders.

If you see 50 "Untitled" items: your query is wrong or empty. Those are likely
junk folders created when Download was set to Folder → Create. Delete them in Drive.

Output: one item per markdown file (id, name, mimeType). Next node downloads each file by id.

OpenRouter app attribution (BCAI Booking in dashboard)
------------------------------------------------------
**Call OpenRouter (Booking)** sends:
  HTTP-Referer: http://178.104.254.165/booking   (unique app URL — must be a URL, not the title)
  X-OpenRouter-Title: BCAI Booking
  X-Title: BCAI Booking

Do **not** put "BCAI Booking" in HTTP-Referer — that header must be your app's URL.
The display name goes in **X-OpenRouter-Title** (and **X-Title**). We also send standard **Referer**.

**Activity log vs app name:** The App column often shows the referer URL (e.g. http://178.104.254.165/booking)
as the app identifier — that means attribution is working. The human-readable name "BCAI Booking" appears on
your app analytics page: https://openrouter.ai/apps?url=http://178.104.254.165/booking

Troubleshooting — blank booking page
------------------------------------
If GET /webhook/booking returns HTTP 200 with an empty white page:

1. **Duplicate workflows** — only one "BCAI Discovery Call Booking" should be active.
   Delete or archive extras in n8n (Workflows list). Webhook path conflicts deactivate registration.

2. **Get Calendar Events outputs zero items** — when your calendar has no upcoming events,
   n8n stops the chain unless **Always Output Data** is ON on that node.
   The repo workflow sets `alwaysOutputData: true` and `onError: continueRegularOutput`.
   Re-import `n8n/bcai-discovery-booking.workflow.json` if you edited nodes manually.

3. **Workflow not published** — toggle Active in the editor, or:
     docker exec n8n n8n publish:workflow --id=bcai-discovery-booking-001
   Then restart n8n if webhooks still 404.

4. **Credentials** — attach Google Calendar OAuth2 on Get Calendar Events + Create Calendar Event,
   and Gmail OAuth2 on Send Confirmation Email after import.
